# 形象素材目录

每个子文件夹是一套完整形象包，程序启动时会**自动扫描** `形象/` 下的子文件夹，无需写死预设列表。

添加或修改形象：

1. 在 `形象/` 下新建或重命名文件夹，例如 `形象/我的角色/`
2. 放入 `.png` 或 `.gif`（透明底推荐 PNG + `CHARACTER_MATTE=off`）
3. 可选：同目录放 `人设.txt`（或 `persona.txt`）绑定 AI 人设
4. **重启程序**（或托盘「切换形象」）即可识别

**重命名文件夹**：若保留同一份 `人设.txt` 内容，程序会按内容指纹自动匹配新文件夹名并更新配置；若只改了文件夹名、删了人设，请用托盘「切换形象」选一次。

**不必固定文件名**。把 PNG / GIF 丢进文件夹即可，程序会**随机轮播**所有动作。

若希望某张图在特定时机播放，请在文件名里带上**用户操作**相关词（自动识别）：

| 场景 | 文件名示例（含任一关键词即可） |
|------|-------------------------------|
| 鼠标移入 | `悬停.png`、`鼠标触碰.gif`、`hover.png`、`pointer.png` |
| 左键拖动窗口 | `拖动.png`、`drag_hold.gif`、`拖拽中.png` |
| 其它 | `打招呼.png`、`跳起来.png`、`动作1.png` → **随机轮播**（动作名不算操作词） |

无需在 `.env` 里写动作列表；换文件夹或改名后**重启程序**即按新文件名重新识别。

## 绑定人设（与形象同文件夹）

在形象包目录里放 **`persona.txt`**（或 `提示词.txt` / `人设.txt` / `prompt.txt`），
程序会优先用这里的文字作为 AI 人设，与这套立绘绑定。

示例结构：

```text
形象/我的小猫/
  动作1.png
  动作2.png
  persona.txt    ← 小猫的说话风格、性格（见 persona.example.txt）
```

切换形象包 = 同时切换立绘 + 人设。模板见 `persona.example.txt`。
